Architecture Guide

Monica follows a feature-oriented architecture designed for scalability, maintainability, and clean separation of concerns.

Project File Structure

The project is organized into several key directories, each with a distinct responsibility.

/
├── components/         # Reusable UI components
├── contexts/           # React context for state management
│   ├── hooks/          # Custom hooks for logic domains
│   └── StateProvider.tsx # Main context provider
├── services/           # External API interactions (Gemini API)
├── types/              # TypeScript type definitions
└── ...                 # Other root files

Data Flow & Logic Separation

The application's architecture ensures a unidirectional data flow and clear separation between different layers of logic.

  1. UI Layer (/components): These are "dumb" components responsible for rendering the UI and capturing user input. They receive data and event handlers as props and have minimal internal logic.
  2. State & Orchestration Layer (/contexts): This is the heart of the application.
    • StateProvider.tsx acts as the central orchestrator, initializing state and assembling the various logic domains from custom hooks.
    • Custom Hooks (/contexts/hooks) each manage a specific domain of logic (e.g., useChatHandler, useConversationManager).
  3. Service Layer (/services): This layer is responsible for all external communication with the Google Gemini API. Services are stateless functions that handle prompt construction, API calls, and error handling.

Example Flow: Sending a Message

  1. User types in MessageInput.tsx and clicks "Send".
  2. The component calls handleSendMessage from the useAppContext hook.
  3. handleSendMessage in useChatHandler.ts updates the state with the user's message.
  4. Based on the mode, it calls a function from the managerService.ts or agentService.ts.
  5. The service function builds a prompt and calls the Gemini API.
  6. The response returns to the service, then to useChatHandler.
  7. useChatHandler updates the state with the new AI message, triggering a re-render.
