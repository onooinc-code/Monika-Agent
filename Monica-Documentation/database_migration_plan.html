Database Migration Plan: from LocalStorage to Cloud-Native

Rationale: Why Migrate?

Migrating from the browser's localStorage is essential for a production-grade application to enable multi-device sync, scalability, security, and powerful server-side features.

Path A: Google Cloud Firestore

A NoSQL, document-oriented database from Google known for ease of use and real-time capabilities.

  • Benefits: Rapid development, real-time by default, serverless integration, generous free tier.
  • Structure: Uses a nested structure of collections and documents, ideal for flexible data like chat messages and user settings.
{
  "users": {
    "{userId}": {
      "profile": { "name": "Alex" },
      "conversations": {
        "{conversationId}": {
          "title": "...",
          "messages": { "{messageId}": { "text": "..." } }
        }
      }
    }
  }
}

Path B: Vercel Postgres

A serverless SQL database that integrates seamlessly with the Vercel ecosystem, offering strong data integrity and powerful querying.

  • Benefits: First-party Vercel integration, full SQL power, strong data integrity, and a familiar relational model.
  • Structure: Uses traditional tables with relationships. The JSONB type is used for flexible, schemaless data (like agent configurations) within a structured table.
-- users Table
CREATE TABLE users (id UUID PRIMARY KEY, ...);

-- conversations Table (Many-to-One with users)
CREATE TABLE conversations (id UUID, user_id UUID, ...);

-- messages Table (Many-to-One with conversations)
CREATE TABLE messages (id UUID, conversation_id UUID, ...);
