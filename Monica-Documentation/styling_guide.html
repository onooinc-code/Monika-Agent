Styling Guide

Monica's visual identity is built on a "Future World" aesthetic, combining dark mode, glassmorphism, and neon highlights to create a modern, immersive user experience. This guide outlines the core principles and tools used to achieve this look.

Core Technologies

  • Tailwind CSS: A utility-first CSS framework is used for all styling, enabling rapid development and a consistent design system.
  • Custom CSS: The shared shared.css file defines CSS variables, animations, and other custom styles that are not easily achievable with Tailwind utilities alone.

Aesthetic Principles

  • Glassmorphism: Key UI panels use a blurred, semi-transparent background (glass-pane class) to create a sense of depth and context.
  • Neon Glows: Interactive elements use a subtle neon box-shadow to draw attention and provide a futuristic feel.
  • Dark Mode First: The entire interface is designed with a dark background to reduce eye strain and enhance the neon highlights.
  • Micro-interactions & Animations: All interactive elements have smooth transitions. New content often appears with a subtle fade-and-slide-in effect.

Key CSS Variables

Colors, fonts, and other core design tokens are defined as CSS variables for easy theming and consistency.

:root {
  --font-sans: 'Inter', sans-serif;
  --color-bg: #0a0a0f;
  --color-pane: rgba(23, 26, 43, 0.7);
  --color-border: rgba(129, 140, 248, 0.2);
  --color-neon-indigo: rgba(129, 140, 248, 1);
}

Animations

Custom keyframe animations are defined globally to be reused across the application.

  • fadeInUp: Used for modals and new messages to have them slide and fade into view.
  • animated-gradient-text / -bg: Creates a shimmering, moving gradient effect used in headers.
